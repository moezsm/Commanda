# Commanda Iterative Tasks

## Purpose

This folder provides execution-ready iterative tasks for implement agents. Each iteration ends with an MVP slice that is testable and deployable.

## How to Use

1. Complete iterations in order unless dependencies allow parallel work.
2. Use each iteration file as the source of truth for scope and acceptance.
3. Do not expand scope inside an iteration without creating a new task in the next iteration.
4. Ship the MVP end state before moving forward.

## Definition of Ready for Any Task

- Requirement is clear and bounded.
- Dependency tasks are complete.
- Acceptance criteria are testable.
- No hidden product decisions remain.

## Definition of Done for Any Iteration

- All must-have tasks complete.
- Acceptance criteria validated.
- No cross-tenant data exposure.
- Mobile usability confirmed for primary flow.
- Iteration MVP end state demonstrable.
